# NotionNext 主题共创长期活动

这是一套常设社区活动方案，用于帮助站长、使用者、设计参与者和主题贡献者长期参与主题设计、AI 辅助开发和文档补充。

它不要求主理人持续手动推进。日常运转应由 Discussions、Issue 模板、任务池、轮值协调员和 Draft PR 自然流动。完整分工见 [主题生态长期运营模型](./THEME_ECOSYSTEM_OPERATING_MODEL.zh-CN.md)。

## 活动目标

- 收集更多主题风格方向。
- 帮助暂不熟悉完整前端开发流程的使用者，用 AI 完成小范围主题改造。
- 把聊天里的想法转成 Discussions、Issue、PR。
- 让首次参与者能从小任务开始，而不是一开始就承担完整主题。

## 推荐周期

建议以 4 周为一个循环，长期重复运行：

| 周期 | 目标 | 产出 |
| --- | --- | --- |
| 第 1 周 | 征集主题风格 | Discussions 主题建议帖 |
| 第 2 周 | 拆解设计与选基准主题 | 任务清单 / Issue |
| 第 3 周 | AI 辅助实现最小版本 | 小 PR 或草稿 PR |
| 第 4 周 | 补文档、截图、预览图 | 可合并 PR |

主理人只需要在第 2 周快速判断优先级，以及在第 4 周处理高影响 PR。其它时间由主题协调员和社区志愿者推进。

## 参与角色

| 角色 | 可以做什么 |
| --- | --- |
| 站长 | 提供参考站点、截图、使用场景 |
| 设计参与者 | 拆解视觉语言、整理组件规范 |
| 首次参与者 | 用 AI 改一个组件或一个页面 |
| 主题维护者 | Review 主题结构、移动端、深色模式 |
| 文档贡献者 | 补主题说明、教程、截图 |
| 主题协调员 | 每月整理建议、拆 Issue、维护任务池 |

## 主理人低负担规则

- 不逐条回复所有主题想法，只回复成熟候选方向。
- 不处理个人站点定制问题，除非能抽象成通用能力。
- 不 Review 大而全草稿，要求拆成小 PR。
- 只对全局架构、数据层、公共组件和长期方向做最终判断。
- 鼓励社区成员先互相答疑，维护者只补文档和纠偏。

## 可认领任务

适合标记为 `good first issue`：

- 补充某个主题的预览截图。
- 补充某个主题的适用场景说明。
- 给主题切换面板补 `summary`。
- 修正主题文档中的配置项说明。
- 给 AI 主题开发教程补一个真实案例。

适合标记为 `help wanted`：

- 基于 `example` 做一个新主题雏形。
- 给现有主题补移动端导航体验。
- 给现有主题补深色模式完整适配。
- 整理一个主题风格方向的参考站点清单。
- 把一个成熟 Discussions 想法拆成 Theme contribution Issue。

```markdown
## 本期主题方向

本期想征集 / 共创的主题方向是：

- 风格：
- 适用站点：
- 参考链接：
- 推荐基准主题：

## 本期任务

- [ ] 收集 3 个参考站点
- [ ] 输出 UI 拆解
- [ ] 复制基准主题并完成首页雏形
- [ ] 补移动端导航
- [ ] 补深色模式
- [ ] 补主题文档
- [ ] 补预览图和 manifest

## 参与方式

只提供参考资料也可以，不要求具备代码能力。
准备尝试 AI 辅助主题改造时，建议先阅读：

- docs/user-guide/development/ai-theme-prompts.md
- docs/user-guide/development/notion-next-develop-with-ai.md
- docs/developer/THEME_MIGRATION_GUIDE.zh-CN.md
```

## 维护者操作清单

1. 在 Discussions 发布“主题风格征集与 AI 辅助开发”帖。
2. 每周把成熟建议整理到 Issue。
3. 给简单任务加 `good first issue`。
4. 给需要协作的任务加 `help wanted`。
5. 鼓励新贡献者先开 Draft PR。
6. PR 合并后在致谢页或 Discussions 中回顾成果。
