# AI 主题开发提示词包

这份提示词包用于降低 NotionNext 主题二次创作门槛。你可以直接复制其中的提示词，配合 GitHub Copilot、Cursor、Firebase Studio、Codex 或其他 AI 编程工具使用。

使用前建议先读：[借助 AI 开发 NotionNext](./notion-next-develop-with-ai.md) 与 [开发自己的主题](./own-theme.md)。

## 使用原则

1. 先让 AI 分析，不要一上来改代码。
2. 一次只改一个页面或组件区域。
3. 每次修改后本地运行并截图检查。
4. 不要直接覆盖其它主题目录里的组件。
5. 准备贡献主仓库时，补齐主题文档、预览图和主题切换配置。

## 1. 拆解参考站点

适合在你看到一个喜欢的网站、主题或截图时使用。

```text
请帮我拆解这个网站/截图的 UI，不要写代码。

请按以下结构输出：
1. 页面整体布局
2. 导航结构
3. 首页信息层级
4. 文章列表或卡片样式
5. 文章详情页可能的布局
6. 色彩、圆角、阴影、边框、间距
7. 字体层级
8. 移动端适配要点
9. 深色模式适配建议
10. 哪些部分适合迁移到 NotionNext 主题，哪些不建议迁移

目标：我想把这个风格做成 NotionNext 的一个新主题。
```

## 2. 选择基准主题

适合在开始复制主题前使用。

```text
我想为 NotionNext 做一个新主题，目标风格是：
【在这里描述风格，例如日式便当卡片、Material 扁平化、极简作品集、SaaS 官网】

请根据 NotionNext 现有主题定位，建议我基于哪个主题开始：
- example
- simple
- endspace
- starter
- gitbook
- landing
- nav

请输出：
1. 推荐基准主题
2. 为什么适合
3. 不推荐哪些主题以及原因
4. 第一阶段只需要改哪些页面
5. 第一阶段不要碰哪些复杂功能

请先不要写代码。
```

## 3. 生成改造清单

适合你已经复制了主题目录之后使用。

```text
我已经把 `themes/example` 复制为 `themes/<你的主题名>`。

请阅读以下文件：
- `themes/<你的主题名>/index.js`
- `themes/<你的主题名>/config.js`
- `themes/<你的主题名>/components/`

目标风格是：
【描述目标风格】

请先不要修改代码。请按优先级输出改造清单：
1. 首页布局
2. 文章卡片
3. 导航和移动端菜单
4. 文章详情页
5. 分类/标签/搜索/404
6. 深色模式
7. 主题配置项
8. 文档与预览图

每一项请说明要改哪些文件、为什么改、验收标准是什么。
```

## 4. 改一个组件

适合让 AI 聚焦在一个小范围，避免一次改太多。

```text
请只修改【组件名或文件路径】，不要改其它文件。

目标：
【例如：把文章卡片改成圆角卡片布局，桌面端左文右图，移动端上下排列】

约束：
- 保留现有 props，不改变数据来源
- 保留深色模式 class
- 无封面图时不能布局塌陷
- 移动端不能横向滚动
- 不直接引用其它主题目录的组件

请先说明修改方案，再给出代码改动。
```

## 5. 补主题配置项

适合把颜色、文案、开关抽到 `config.js`。

```text
请把当前主题中适合给站长调整的内容抽到 `themes/<主题名>/config.js`。

要求：
- 使用清晰的配置名
- 保留默认值
- 组件中通过现有 `siteConfig()` 读取
- 不要把站点私有内容硬编码在组件里

请先列出建议新增的配置项，再修改代码。
```

## 6. 检查主题质量

适合提交 PR 前使用。

```text
请以 NotionNext 主题 PR reviewer 的角度检查当前主题改动。

重点检查：
1. 是否破坏现有数据契约
2. 是否直接耦合其它主题组件
3. 首页、文章页、列表页、搜索页、404 是否覆盖
4. 移动端是否可用
5. 深色模式是否完整
6. 无封面、无摘要、无标签等空数据是否安全
7. 是否需要补充 `docs/user-guide/themes/<id>.md`
8. 是否需要补充预览图和 `conf/themeSwitch.manifest.js`

请按 P0/P1/P2 列出问题，并给出具体文件路径。
```

## 7. 写主题文档

适合完成主题初版后使用。

```text
请为 NotionNext 新主题 `<主题名>` 生成用户向文档草稿。

请包含：
1. 主题简介
2. 适用场景
3. 主题预览链接格式
4. 如何启用主题
5. 主要配置项
6. 适合哪些站点
7. 已知限制
8. 相关链接

文档目标路径是：`docs/user-guide/themes/<id>.md`。
语气要面向普通站长，不要写成开发者实现说明。
```

## 8. 发起社区讨论

适合你还没有开始写代码，只想先收集反馈。

```text
请帮我把下面的主题想法整理成 GitHub Discussions 帖子。

要求：
- 标题清晰
- 说明目标用户
- 说明参考风格
- 列出希望大家反馈的问题
- 说明如果有人愿意参与，可以从哪些小任务开始

主题想法：
【粘贴你的想法】
```

## 最小可行任务

第一次参与建议只做这些小任务之一：

- 给一个现有主题补移动端截图。
- 给一个主题补 `docs/user-guide/themes/<id>.md` 的适用场景。
- 基于 `example` 改一版文章卡片，不改文章页。
- 给一个主题补深色模式缺失样式。
- 给一个主题补 `themeSwitch.manifest.js` 的简介。
